Software Tools

As mentioned above, we will be using Arduino IDE 1.8.13 to program the sensor probe. This is an open-source project however it is not free for commercial use unless you make your object files public in the Arduino library as per the LGPL license. As such, The Green Thumb project will be made public in the Arduino library.

Additionally, to develop the probe prototype we will use the Blynk IoT platform to be able to control the probe from a mobile device. It is free to use this software in the prototyping stage but credits must be purchased to add informative, interactive or static widgets onto the app’s user interface. We will need to use 3 widgets for the prototype to sense moisture, temperature and acidity. We purchased 28 thousand credits for $35 which was spent towards these widgets.

To get the Green Thumb app ready for the market, we will need to build our interface from scratch rather than using Blynk which is mostly intended for testing and prototyping functions/use cases with very limited features and options to use. Building the Green Thumb app from scratch is out of scope at this stage, so our primary focus during assignment 3 will be building a prototype of the app. To do this we will use Figma prototyping software. Figma has a free starter plan that includes:

  • Unlimited files in drafts 
  • Unlimited viewers and commenters 
  • Unlimited editors on 3 team files 
  • 1 team project 
  • 30-day version history 
  • Unlimited cloud storage 

This plan will suffice for our needs. Subject to the terms and conditions, Figma grants a limited, non-exclusive, non-transferable, non-sublicensable license worldwide. Within our team, Bridget has some limited experience using Figma. She has worked within teams to develop website prototypes on Figma. She will be able to guide our UI/UX team to create a user-friendly app design.

When we are finally ready to develop the market-ready Green Thumb app we will use Swift for iOS and/or generic Java IDEs for Android as these options are the most widely used and have the most support either by the companies themselves or via the community. No members of our group have experience programming with these softwares so we will seek advice from professionals.

Green Thumb relies on data to perform, and as such we will need to build an expansive database to store customer data, plant data, and machine learning data. No members of ABRAIN have experience programming databases, so building a whole database from scratch is out of scope. Instead of building an entire database, we will use Lucidchart to map out an entity-relationship diagram and generate the code required to build a database. Lucidchart has a free plan that includes:

  • 3 editable documents 
  • Templates 
  • Multiple collaborations 
  • Basic integrations  

On the free plan, the Lucidchart licence only enables one user to build diagrams. The rest of the team members will be granted view and comment-only licenses so they can give feedback on the work but not edit it. This is a slight limitation but we will be able to work around this. Once our database is concepted we will use Microsoft SQL Server (MS SQL) to build and host our database. MS SQL uses the same Structured Query Language (SQL) that alternative solutions, such as MySQL, also use. We will be utilising MySQL to host the database of the computer while also using MS SQL to create, manage and access databases on both local and remote machines.

Finally, the Green Thumb AI will be built within IBM Watson Studio. Within Watson Studio we will utilize the Image Recognition, Machine Learning, AutoAi and Chatbot features to bring the Green Thumb AI to life. No members of our team have experience with this software, however Renee and Bridget have both created accounts and have a basic idea of how the software works. We will need to engage with machine learning specialists to deploy the AI. To deploy the AI we will need to set up a monthly subscription to IBM Watson Studio that will grant us a perpetual license to use the software.
