Will You Be Introducing Any Changes in Process For Assignment No. 3?

There were unforeseen circumstances that happened for A2 and could have been prevented if we well planned it. In this case, we have introduced several changes for Assignment No. 3 so we can make this time’s work even better. They are:
Having fixed regular meetings twice a week, rather than scheduling random days to hold the gathering
Making sure everyone is present at the designated time slot we assigned for that day/meeting, instead of waiting until all members are present, so we can make the most out of it and avoid extensions when not necessary
Setting smaller deadlines to large tasks to make them easier to tackle. Additionally, team members will work harder to meet these deadlines to ensure work doesn’t get left until the last minute
Allowing more time to work on the website portion of this assignment, especially when transferring the content through the GitHub. In this case, we start on putting them at Week 11, so if there are any unpredictable coding changes or accidental mistakes there is time allocated to fix it and not struggle at the near-end
A more collaborative approach to work rather than members working in isolation. Also Comments will be openly-available if they need to state something from every member’s work/contribution, because feedback to each other is very important.
